3. DÉCISIONS D'ARCHITECTURE
Lecture audio
Podcasts RSS  →  stream direct CDN  →  ExoPlayer  →  seek byte-range instantané
Livesets YT   →  download yt-dlp    →  fichier MP3 local  →  ExoPlayer  →  seek local 0ms

Pourquoi : URLs YouTube expirent en 4-6h. Byte-range seek non garanti sur streams YouTube. Les podcasts RSS sont sur CDN stables — pas besoin de download.

Timestamps — sources validées par test
Priorité 1 : YouTube chapters  (yt-dlp info['chapters'])      → secondes exactes
Priorité 2 : YouTube description (yt-dlp info['description']) → parser MM:SS
Priorité 3 : MixesDB API MediaWiki                            → [H:MM:SS] ou [MM]
Priorité 4 : YouTube comments  (yt-dlp info['comments'])      → [MM:SS] dans texte
Priorité 5 : ShazamIO sur fichier local                       → lent, dernier recours
MORT       : 1001Tracklists → Cloudflare Turnstile + AJAX
MORT       : Mixcloud API sections → OAuth requis

4. PLAN DE DÉVELOPPEMENT — SÉQUENTIEL
Phase 1 — Pipeline timestamps Python (backend local)
Input : URL YouTube d'un liveset
Process :

yt-dlp.extract_info(url)
  → chapters?      → parse direct → List[{start_sec, artist, title}]
  → description?   → regex MM:SS  → List[{start_sec, artist, title}]
  → MixesDB API    → search(titre+artiste) → wikitext → parse [H:MM:SS]
  → comments?      → top comment avec ≥5 lignes timestampées → parse

Output : JSON [{"start": 847, "artist": "Alignment", "title": "Blade"}]
Fichier : audio_timestamp_server.py — remplacer le stub /analyze

Phase 2 — Download livesets via yt-dlp
Input : Episode avec youtubeVideoId ou mixcloudKey
Process : yt-dlp -x --audio-format mp3 --write-info-json → dossier local Android
Output : liveset_ID.mp3 + liveset_ID.info.json (timestamps inclus dans info.json)
Fichier : nouveau service Android LivesetDownloadService.kt

Phase 3 — Fix PlayerController URL expirée

Fichier : PlayerController.kt:104-116
